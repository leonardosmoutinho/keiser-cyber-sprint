# Hardware Deployment Process

## Overview

20+ desktop computers were assembled, configured, and deployed from scratch as part of the Seahawk Cyber-Sprint lab build.

## Assembly Process (Per Unit)

1. **Physical inspection** — check chassis integrity, existing components
2. **Component installation** — RAM seating, CPU cooler mounting, storage drive installation
3. **BIOS/UEFI configuration** — boot order, hardware detection, date/time
4. **OS deployment** — bootable USB creation, Windows 10/11 or Linux Mint installation
5. **Post-install verification** — drivers, updates, network connectivity test
6. **Documentation** — unit recorded in inventory log

## Operating Systems Deployed

| OS | Use Case |
|---|---|
| Windows 10/11 | Primary student workstations |
| Linux Mint | Linux/open-source lab stations |

## Hardware Tested Per Unit

- RAM compatibility and seating
- Storage drive detection (HDD/SSD)
- CPU cooler contact and thermal paste
- Power supply and boot sequence
- Network adapter functionality
- Display output

## Tools Used

- Bootable USB drives (Rufus/Balena Etcher)
- ESD wrist strap (anti-static protection)
- Multimeter (voltage/continuity checks)
- Standard screwdriver set
- Magnifying tool (component inspection)
